## 3. 升级后异常

```md
这类问题先不要直接按经验处理，先核对升级路径和附加操作。

建议顺序：
1. 到 [版本页](https://docs-pd.mingdao.com/version) 确认你的目标版本和发布日期
2. 打开对应的升级详情页检查是否有附加操作
3. 再对照当前部署模式的升级文档核验实际执行步骤
```

## 5. 工作流持续排队

```md
官方 FAQ 已覆盖这个场景。

问题判断：
先区分是"排队量大但仍在消费"，还是"一直排队完全不消费"。

官方页面：
- [工作流持续排队](https://docs-pd.mingdao.com/faq/troubleshooting/workflow-keeps-queuing/)

建议执行顺序：
1. 看工作流监控页，确认是否有单个流程排队数异常高。
2. 如个别流程数十万/百万级排队，先判断是否触发逻辑错误、死循环或批量触发过大。
3. 如全部不消费，检查磁盘：`df -Th`。
4. 检查 Kafka 或存储组件健康日志。
5. 检查 Kafka 消费组 `md-workflow-consumer` 的 `LAG` 和是否长时间 `rebalancing`。
6. 只有在消费组长时间 rebalancing 或根因排查后，再重启工作流消费服务。

未确认项：
- HAP 版本、部署方式、排队是否还在下降、Kafka 消费组输出。
```
